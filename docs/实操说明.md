# DeskCar v2 自动回冲 · 完整实操说明

> 目标：把一台 ESP32 小车、一台桌面摄像头、一个 Qi 充电坞组装起来，让 PC 端在摄像头的引导下把车开回充电坞并自动充电。全部步骤按从上电到首次跑通自动回冲的真实执行顺序排列。

如果你只是想确认这套东西的逻辑能不能跑通，可以跳过阶段 1~6，直接看 `orchestrator/tests/test_auto_recharge.py` 跑端到端冒烟测试。要把整套东西搬到桌面上跑起来，按下面的顺序来。

## 阶段 1  硬件搭建

### 1.1 打印 3D 件

打开 [hardware/](../hardware/)：

- `hardware/dock/`：底座、垂直 pad 支架、tag 连杆、tag 安装板
- `hardware/chassis_v2/`：RX 支架、ArUco 立柱
- `hardware/mount/`：摄像头支架（1/4"-20）

打印完成后通孔攻丝 M3。

### 1.2 组装小车

按 `docs/HARDWARE.md` 第 3 节的引脚表接线：

- DRV8833 → 两个 N20 电机（PWM 50 ms 缓启斜坡已在固件里）
- INA219 → I2C 0x40（SDA=21 / SCL=22）
- BQ51025B → CHRG=25、STBY=33（如果有）
- 18650 + TP4056 + DW01A
- 5 针 pogo 扩展口（3.3V / GND / SDA / SCL / INT=26）
- 加 470 µF + 100 nF 退耦到 DRV8833 的 VM

**车顶 ArUco 立柱**：把 32 mm 边长、`DICT_4X_4_50` ID=0 的 ArUco 码**朝上**贴在立柱顶面；摄像头看到的就是这张码的姿态。

### 1.3 组装充电坞

- 5V/3A USB-C PD → XKT-510 / BQ500100 → 40 mm TX 线圈
- 垂直 pad 支架把 TX 线圈**竖直**立在桌面上
- 侧标连杆从 pad 朝外法线方向**向左**伸出 40 mm
- tag 安装板贴 60 mm 边长 `tag36h11` ID=0 AprilTag，**"上边缘"朝向 pad**（如果装反，把 `dock_pad_offset_m` 改成 `-0.04` 即可）

### 1.4 固定摄像头

把 USB 摄像头装在 `mount/` 支架上，镜头**正对**工作区域中心，距离桌面 70 cm 左右。FOV 90° 足以覆盖 1.4 m × 0.8 m 的桌面。

### 1.5 上电

- 小车上电：开机会建一个名为 `ESP32_Car_Control` 的开放 Wi-Fi 热点
- 充电坞上电：TX 线圈 LED 亮

## 阶段 5  标定摄像头（单应矩阵）

把车挪开，确保桌面上**只**有 4 个明显的角点（用胶带贴也行）。然后跑标定脚本：

```bash
cd ../../orchestrator
python scripts/calibrate_homography.py --width 1.4 --height 0.8 --camera 0
```

弹出画面后**按顺序**点击桌面的 4 个角：

1. 左上 → 世界坐标 `(0, 0)`
2. 右上 → `(width, 0)`
3. 右下 → `(width, height)`
4. 左下 → `(0, height)`

脚本会写出 `orchestrator/deskcar_orch/calib/homography.npy`。`runtime.py::_load_homography_or_identity()` 会在每次启动时自动加载它。

> **没有 OpenCV 摄像头时**：见 `docs/CALIBRATION.md` 2.1 节的纯 numpy DLT 替代方法。

## 阶段 6  测量 tag → pad 偏移

`configs/default.yaml` 默认值是按 `HARDWARE.md` 的尺寸给的：

```yaml
charger:
  dock_pad_offset_m: [0.04, 0.0]   # tag 中心 → pad 中心 +x = 40 mm
  dock_pad_stand_off_m: 0.08      # 车前脸离 pad 面 80 mm
```

装好之后做一次实测：

1. 把车开到 tag 视野内，PC 端读 `charge=charging`（假装它已经在充）
2. 观察日志里打印的 car pose 与 dock pose
3. 用直尺量出实际偏差，改 `dock_pad_offset_m` 覆盖

这一步需要试 2~3 次，每次只改一两厘米。改完不用重启固件，只重启 orchestrator 即可。

## 阶段 7  第一次自动回冲

### 7.1 强制触发

```bash
python -m deskcar_orch -c configs/default.yaml --dock -v
```

参数：

- `-c`：配置文件
- `--dock`：跳过电量判断，立刻进入回冲
- `-v`：开 DEBUG 日志，看每一帧的 pose、twist、状态机

### 7.2 期望看到的状态机顺序

```
IDLE  -> (BATTERY_LOW / force_dock)  -> SEEK_DOCK
SEEK_DOCK -> (AprilTag 可见)         -> ALIGN
ALIGN  -> (heading 收敛)            -> APPROACH
APPROACH -> (距离 0.05 m 内)         -> COUPLE
COUPLE -> (charge=CHARGING 持续 2 s) -> CHARGING
CHARGING -> (SoC >= 95 % / full)    -> FULL
FULL -> (USER_UNDOCK)               -> UNDOCK
UNDOCK -> (后退 0.20 m / 超时)      -> IDLE
```

### 7.3 调一些可能用得着的参数

`configs/default.yaml` 里这几个最常调：

| 字段 | 默认 | 调整时机 |
| :--- | :--- | :--- |
| `charger.dock_pad_offset_m` | `[0.04, 0.0]` | 实测不准时 |
| `charger.dock_pad_stand_off_m` | `0.08` | 车没对齐 pad 时 |
| `charger.couple_settle_s` | `2.0` | 接触 pad 抖动时调小 |
| `charger.dock_soc_threshold` | `20.0` | 想早回冲就调高 |
| `servo.kp_linear` / `servo.kp_angular` | `0.8 / 2.5` | 接近 dock 时振荡 |
| `servo.max_linear_mps` | `0.30` | 小车机械速度上限 |

### 7.4 验证回冲成功

成功标志：

- 日志里 `INFO  charge coupled after X s`
- 状态从 COUPLE → CHARGING
- `state.charge` 在 1 秒内变成 `charging`
- INA219 电流 `i` 变负（充电中）
- 5~10 分钟后 `state.charge` 变成 `full`

## 阶段 8  触发条件：低电量自动回冲

阶段 7 是手动 `--dock` 触发的。要让车**自己**因为电量低去回冲：

- 确认 `configs/default.yaml` 里 `charger.dock_soc_threshold: 20.0`
- 把车放到桌面其他位置，**不**给 `--dock`
- 让车持续运行 `python -m deskcar_orch -c configs/default.yaml`
- 等电量降到 20 % 以下

或者临时把阈值调高来快速验证：

```yaml
charger:
  dock_soc_threshold: 80.0
```

跑起来后状态机应该从 IDLE 直接进 SEEK_DOCK。

## 阶段 9  常见调参与故障排查

| 现象 | 大概率原因 | 处理 |
| :--- | :--- | :--- |
| `discover failed` | 笔记本没连上小车的 AP | 检查 Wi-Fi；用 `--host 192.168.4.1` 跳过发现 |
| 标定矩阵太大，pose 跳变 | 角点顺序错了 | 重跑阶段 5，按 TL→TR→BR→BL 顺序点 |
| ALIGN 阶段抖动 | `kp_angular` 太大 | 调到 1.5 左右 |
| APPROACH 阶段离 dock 还差几 cm 就停 | `dock_pad_offset_m` 偏小 | 实测后 +1~2 cm |
| COUPLE 阶段一直 drive 但不 dispatch COUPLED | charge 信号没接好 / INA219 没读到 | 看日志 `last_state`，确认 `i` 正常 |
| 自动回冲整圈都过不了 | 摄像头被遮挡 | 挪开手 / 调高 mount |
| 充电坞上没电流 | 线圈没对齐 | 用尺量 RX/TX 中心偏差，控制在 5 mm 内 |

排错时把所有日志打开：

```bash
python -m deskcar_orch -c configs/default.yaml --dock -v 2>&1 | tee last_run.log
```

`INFO` 级别会打印状态机转换，`DEBUG` 会打印每帧 pose + twist。

## 阶段 10  日常使用

1. 每次开机确认小车上电、热点出现
2. PC 端连上 `ESP32_Car_Control`
3. `python -m deskcar_orch -c configs/default.yaml`（不加 `--dock`，让它按 SoC 阈值自动决策）
4. 不用时 Ctrl-C 关掉
5. 想强制回冲：加 `--dock`；想强制离坞：发 `USER_UNDOCK` 事件（demo 见 `examples/`）

长期建议：

- 把 `orchestrator` 跑在 Windows 自启动 / systemd service 里
- `charger.dock_soc_threshold` 调到你常用的低电量值（建议 30 %）
- 每次换了摄像头 / 桌子 / 挂架，**重新跑一次**阶段 5 的单应标定
- 固件 / SDK / orchestrator 任何一边动协议，先跑 `sdk/tests/test_wire_shape.py` 验证一遍
