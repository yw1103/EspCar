# DeskCar v2 硬件施工方案

> 本方案面向 3D 建模/打印，覆盖“自动回充”所需的最小硬件改动：充电仓（含 AprilTag 侧标）、车体 RX 线圈支架、电源与信号约束。

## 1. 设计取舍

* **不做避障**，因此车体结构保持 v1 底盘不变。
* **无线充电板垂直放置**，小车正面带 RX 线圈，倒车/正车入仓均可，但本方案采用**正面驶入**。
* **AprilTag 贴在充电板左侧的水平板子上**，从摄像头视角看无遮挡，直到小车与充电板贴合。
* tag 中心到 pad 中心的偏移仅增加一次 2D 刚体旋转（`runtime.py::compute_pad_target`），对 ESP32/PC 计算量可忽略。

## 2. 坐标与尺寸约定

所有打印件尺寸单位：mm。

| 参数 | 默认值 | 来源 |
|------|--------|------|
| tag 中心 → pad 中心偏移 | `+x = 40 mm` | `../orchestrator/configs/default.yaml` 中 `charger.dock_pad_offset_m` |
| 停车时车前脸距 pad 面 | `80 mm` | `charger.dock_pad_stand_off_m` |
| tag 边长 | `60 mm` | `vision.dock_tag_size_mm` |
| 车顶 ArUco 标定码边长 | `32 mm` | `vision.car_marker_size_mm` |
| RX/TX 线圈中心离地高 | `30 mm` | 与车轮半径匹配 |

> **tag 方向约定**：tag 的“上边缘”必须朝向 pad，这样默认 `+x=40 mm` 才指向 pad 中心。若实际装反，把配置里的 `dock_pad_offset_m` 改成 `-0.04` 即可。

## 3. 充电仓（Dock）3D 结构

打印件清单：

1. **底座（base_plate）**：`120 x 80 x 6`，四角留 M3 沉头孔。
2. **垂直支架（pad_holder）**：`50 mm` 高，用于把 Qi TX 线圈/隔磁片竖直固定。
3. **侧标连杆（tag_arm）**：从 pad 面朝外法线方向**向左**伸出 `40 mm`。
4. **tag 安装板（tag_plate）**：`70 x 70 x 2`，中心贴 60 mm 边长的 `tag36h11` ID0 AprilTag。

### 3.1 正视图

```text
          摄像头视角（从上往下俯视）
                 ^ 桌面 Y
                 |
                 |
    +------------+------------+
    |            |            |
    |   AprilTag |   垂直     |
    |   (侧标)   |   TX pad   |
    |   60x60    |            |
    +------------+------------+
          <-- 40 mm -->
                 |
                 +--> 桌面 X（pad 法线向外）
```

### 3.2 侧视图

```text
         TX 线圈垂直面
        |<--- 50 mm --->|
        +---------------+  --- 桌面
        |               |   ^
        |   TX coil     |   | 30 mm 离地
        +---------------+   v
        |    底座       |
        +---------------+
```

### 3.3 装配关系

* 侧标板与 pad 面**共面**（同一块竖板的左侧折弯/加接一段水平板）。
* tag 板上表面朝上，面向摄像头；tag 中心与 pad 中心在同一水平线上。
* tag 板水平放置，因此它不会随着小车靠近 pad 而被遮挡。

## 4. 小车改动

### 4.1 前脸 RX 线圈支架

在前保险杠/车架正面增加一个厚度 `2 mm` 的竖板，内嵌 RX 线圈与隔磁片。

```text
      小车正视图
    +-----------------+
    |    RX 线圈窗    |  <-- 线圈中心离地 30 mm
    |   +---------+   |
    |   |  coil   |   |
    |   +---------+   |
    +-----------------+
```

### 4.2 新增传感器/信号

| 信号 | ESP32 GPIO | 说明 |
|------|------------|------|
| `PIN_CHARGE_CHRG` | GPIO 25 | 可选。若充电模块有 `CHRG` 开漏输出，则低电平表示正在受电 |
| `PIN_I2C_SDA` | GPIO 21 | INA219 电流/电压监测 |
| `PIN_I2C_SCL` | GPIO 22 | INA219 电流/电压监测 |
| `PIN_BTN_UNDOCK` | GPIO 4 | 主动退出充电仓按键（下拉/上拉按实际模块） |
| `PIN_STATUS_LED` | GPIO 13 | 状态 LED |

> GPIO 25/26 在本案中用作输入/上拉，未复用 DAC；仍继承 v1 文档“不要优先占用 25/26”的精神，但充电检测需要固定引脚。

### 4.3 顶部磁吸扩展口（5-pin）

为后续开发者预留的 pogo 触点，建议布局：

```text
      车顶俯视图
    +3.3V  GND+
    |         |
    + SDA SCL +
    |   INT   |
    +---------+
```

* 3.3V / GND / I2C SDA / I2C SCL / INT
* 与 ESP32 的同一 I2C 总线共享；INA219 会在 `0x40` / `0x41` / `0x44` / `0x45` 中自动探测，扩展模块需避开实际探测到的地址。

## 5. 电源拓扑

继承 v1 的“双轨 + 星型共地”方案，并加入无线充电与电量监测。

```text
小车侧：
1S Li-ion 3.7V ──► Boost 6V ──► DRV8833 VM
                  │
                  └──► Buck 3.3V ──► ESP32 / 编码器 / 逻辑
                  │
                  └──► BQ51025B 整流后给电池充电（恒压 4.2V）

充电仓侧：
USB-C PD 5V/3A ──► Qi TX 模块 ──► TX 线圈
```

### 关键避坑点

1. **电机电源 6V 轨与电池充电轨独立**：BQ51025B 输出直接接电池，不要接到 6V 电机轨。
2. **电机启动退耦**：DRV8833 `VM` 入口并 `470 µF ~ 1000 µF` 电解电容。
3. **编码器 VCC 与 ESP32 同 3.3V 源**，防止倒灌电锁死芯片。
4. **单点星型共地**：电池负、Boost 负、Buck 负、DRV8833 GND、ESP32 GND、编码器 GND、INA219 GND 必须在一点汇合。
5. **充电成功判定优先用 INA219**：INA219 必须串在电池主回路上，接线方向约定为“电池放电时电流为正、充电时电流为负”。如果实测符号相反，交换 INA219 的 `VIN+`/`VIN-`。
6. **CHRG 引脚是可选增强**：若模块提供 `CHRG`，ESP32 内部 `INPUT_PULLUP` 即可检测，不要外接上拉；若没有这根线，固件会用 INA219 的负电流判断正在充电。

## 7. 装配步骤

1. 打印底座、垂直支架、侧标连杆、tag 板、小车 RX 线圈窗。
2. 将 TX 线圈与隔磁片装入垂直支架，支架固定到底座。
3. 把 60 mm AprilTag ID0 贴在 tag 板中心，tag 板上边缘朝向 pad；用连杆把 tag 板固定到底座左侧，使 tag 中心与 pad 中心水平距离 `40 mm`。
4. 将 RX 线圈与无线充电接收模块装到小车前脸窗内；如果模块有 `CHRG`，可接到 GPIO 25，没有则留空。
5. 将 INA219 串入电池主回路，保证放电为正、充电为负；再把 I2C 接到 ESP32。固件会自动尝试 `0x40` / `0x41` / `0x44` / `0x45`，上电串口会打印实际地址。
6. 接入电机、编码器、电源轨（按 v1 pin mapping）。
6. 整车与充电仓共用一个桌面摄像头，摄像头俯视桌面，能看到完整工作区。

## 8. 打印建议

* 底座建议 `4` 层壁厚、`30%` 填充，保证不翘边。
* 线圈窗/支架留 `0.2 mm` 间隙，方便线圈嵌入。
* tag 板表面喷哑光漆后再贴 AprilTag，减少反光误检。
